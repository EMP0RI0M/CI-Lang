## 1. Core Philosophy: Why Chaos?
CI-Lang exists to formalize a new model of computation based on chaos, entropy, emergence, and nondeterministic evolution. Traditional programming assumes:
- Same input → Same output (Rigid Order)
- Order is free, noise is error.

**Chaos Intelligence assumes:**
- Order is energetically expensive. Chaos is energetically natural.
- Same input → Evolving distribution of outputs.
- Noise is the engine of exploration; entropy is the measure of potential.

### The Landauer Principle
We align with the physics of information. Erasure of information (reducing entropy) costs energy. By computing *with* chaos, we aim for the most energy-efficient intelligence possible.

## 3. CI-Lang 2.0: The Living Language Specification
CI-Lang 2.0 (Codename: **Xaos**) introduces native support for thermodynamics.

### Key Primitives:
- `agent { ... }`: The primary unit of computation. Stateful, autonomous, drifting.
- `state { ... }`: Variables subject to thermodynamic volatility.
- `volatility = f(E)`: Defines how fast a state drifts based on system entropy `E`.
- `update(dt) { ... }`: The chaotic evolution rule for the agent.
- `entropy::measure(S)`: Built-in estimator for swarm disorder.
- `execute_low_entropy { ... }`: Blocks where high-energy order is enforced.

## 4. FluxVM Architecture: The Entropy Kernel
The **FluxVM** is a specialized virtual machine designed for high-density chaotic agents.

### The Entropy Kernel
Unlike classical VMs, FluxVM maintains a global **Entropy Register**.
- **The Boiling Point**: High entropy ($E \rightarrow 1$) causes rapid drift and high divergence.
- **The Freezing Point**: Low entropy ($E \rightarrow 0$) forces deterministic execution.

### Opcodes list (v2.0):
- `STORE`/`LOAD`: Handle `MemoryCell` objects with value and volatility.
- `SET_VOL`: Adjusts the volatility of a variable.
- `AGENT_DEF`: Registers an agent template.
- `SPAWN`: Instantiates agents into the global swarm.
- `ENTROPIZE`: Injects controlled noise into the stack top.

## 5. The Living Memory Model: Alive Memory & Drift
Memory in CI-Lang 2.0 is "Alive". Every variable $v$ evolves according to:
$$v_{t+1} = v_t + \mathcal{N}(0, \omega \cdot E)$$
Where:
- $v_t$ is the current value.
- $\omega$ is the variable's specific volatility.
- $E$ is the global system entropy.

This creates a state space that naturally "vibrates," allowing agents to explore basins of attraction without being explicitly programmed to do so.
